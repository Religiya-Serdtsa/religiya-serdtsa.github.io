## 왜 새 파이프라인이 필요했나요?

바보쉽 예제처럼 거대한 데이터 파이프를 쓰기에는 깃허브 페이지가 너무 가볍습니다. 서버나 DB 없이도 탐색 가능한 블로그가 필요했고, **cwist의 SString 유틸리티**를 그대로 Wasm으로 옮겨서 마크다운 문자열을 안전하게 다루도록 했습니다.

- Emscripten으로 컴파일된 커널이 브라우저 메모리에서 바로 동작합니다.
- cwist의 오류 컨테이너를 그대로 써서 실패 시 메시지를 JSON 없이도 추적합니다.
- md4c를 함께 빌드해 GitHub 호환 문법과 테이블, 체크리스트, 수학 스팬을 지원합니다.

## 깃허브 블로그에 최적화

모든 포스트는 `posts/<카테고리>/*.md`에 저장되며, 헤더에 노출되는 카테고리 역시 폴더 구조를 따릅니다. `categories.cfg`에서 각 폴더의 메타데이터(제목, 설명, 강조 색상, 순서)를 정의하고 GitHub Actions가 `docs/` 산출물을 배포합니다. 필요한 경우 YAML 프런트 매터를 포함하면 정적 제너레이터가 자동으로 HTML을 갱신합니다.

## 한눈에 보는 특징

1. **DB 없이도** 파일만으로 컨텐츠를 관리
2. **cwist 런타임**을 Wasm으로 재사용해 C 코드 자산을 그대로 살림
3. **머터리얼 UI**의 레이아웃과 주황색 하이라이트로 브랜드 통일
4. **다크/라이트 토글**과 로컬스토리지 기억 기능

필요하면 이 포스트를 복제해 새로운 슬러그 이름으로 저장하면 됩니다. 파일만 추가하면 바로 블로그가 새 섹션을 감지합니다.
